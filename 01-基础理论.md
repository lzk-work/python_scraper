# Python 爬虫入门 — 从 SpringBoot/Vue 视角理解

## 1. 爬虫本质 = 反向的前端

你写 Vue 时的流程：
```
浏览器 → 发 HTTP 请求 → 后端返回 JSON/HTML → 前端渲染展示
```

爬虫的流程：
```
Python → 伪装成浏览器发 HTTP 请求 → 拿到 JSON/HTML → 解析提取数据 → 存下来
```

**一模一样，只是"浏览器"换成了"Python脚本"。**

你用 SpringBoot 写过 Controller 返回 JSON，爬虫就是反过来——用 Python 去请求别人的 Controller，然后解析返回的 JSON。

## 2. HTTP 基础（你已会，回顾一下）

### 请求的组成

```
GET /search?q=蓝牙耳机 HTTP/1.1          ← 请求行（方法 + 路径 + 参数）
Host: www.ozon.ru                        ← Headers（一堆键值对）
User-Agent: Mozilla/5.0 ...
Accept: application/json
Cookie: session_id=abc123

                                          ← Body（GET 没有，POST 有）
```

你写 SpringBoot 时：
- `@GetMapping("/search")` → 对应 GET 请求
- `@RequestParam("q")` → 对应 URL 参数 `?q=xxx`
- `@RequestHeader("User-Agent")` → 对应 Headers
- `@RequestBody` → 对应 Body

爬虫就是用 Python 构造这些内容，发出去。

### 响应的组成

```
HTTP/1.1 200 OK                          ← 状态码
Content-Type: application/json           ← Headers

{"products": [...]}                      ← Body（你要的数据）
```

状态码你肯定熟：
- **200** → 成功
- **301/302** → 重定向
- **403** → 禁止访问（反爬说"我不认识你"）
- **404** → 不存在
- **429** → 请求太快，限流
- **500** → 服务器报错

### 两种响应格式

| 格式 | 来源 | 解析方式 | 类比 |
|------|------|----------|------|
| **JSON** | API 接口 | `resp.json()` → dict | 前端调后端接口拿到的 |
| **HTML** | 网页 | BeautifulSoup 解析 | 浏览器渲染的页面源码 |

**优先找 JSON**（更干净、更稳定）。HTML 解析是退路。

## 3. Python 爬虫三件套

### requests — 发 HTTP 请求

```python
import requests

# 最简单的 GET 请求（就像浏览器地址栏输入 URL 回车）
resp = requests.get("https://www.ozon.ru/search/?text=耳机")

# 带 Headers（告诉服务器"我是浏览器不是脚本"）
headers = {
    "User-Agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) Chrome/125.0"
}
resp = requests.get("https://www.ozon.ru/search/?text=耳机", headers=headers)

# 带参数（等价于 ?text=耳机&page=2）
params = {"text": "耳机", "page": 2}
resp = requests.get("https://www.ozon.ru/search/", params=params)

# 看结果
print(resp.status_code)  # 200
print(resp.text)         # 响应正文（字符串）
print(resp.json())       # 如果是 JSON，直接转 dict
print(resp.headers)      # 响应头
```

**类比 SpringBoot：**
```java
// 你在 Vue 里写的 axios 调用
axios.get("/api/search", { params: { text: "耳机", page: 2 } })
// Python 里就是
requests.get("https://xxx/search/", params={"text": "耳机", "page": 2})
```

### BeautifulSoup — 解析 HTML

HTML 是树形结构，像这样：
```html
<html>
  <body>
    <div class="product-card">
      <h2>蓝牙耳机 Pro</h2>
      <span class="price">299 ₽</span>
      <span class="rating">4.8</span>
    </div>
    <div class="product-card">
      <h2>无线耳机 Lite</h2>
      <span class="price">149 ₽</span>
      <span class="rating">4.5</span>
    </div>
  </body>
</html>
```

BeautifulSoup 就是让你用 CSS 选择器（像 jQuery/Vue 的 querySelector）去提取数据：

```python
from bs4 import BeautifulSoup

soup = BeautifulSoup(resp.text, "html.parser")

# 找单个元素 — 类似 document.querySelector(".price")
price_el = soup.select_one(".price")
print(price_el.text)  # "299 ₽"

# 找多个元素 — 类似 document.querySelectorAll(".product-card")
cards = soup.select(".product-card")
for card in cards:
    name = card.select_one("h2").text     # "蓝牙耳机 Pro"
    price = card.select_one(".price").text  # "299 ₽"
    rating = card.select_one(".rating").text  # "4.8"
```

**类比 Vue：**
```javascript
// Vue 里你这样操作 DOM
document.querySelectorAll(".product-card").forEach(card => {
    const name = card.querySelector("h2").textContent
    const price = card.querySelector(".price").textContent
})
// Python 里一模一样的思路
```

## 4. 爬虫的核心套路（只有 4 步）

```python
# 第 1 步：构造请求
url = "https://www.ozon.ru/search/"
params = {"text": "蓝牙耳机", "page": 1}
headers = {"User-Agent": "Mozilla/5.0 ..."}

# 第 2 步：发请求
resp = requests.get(url, params=params, headers=headers)

# 第 3 步：解析响应
data = resp.json()  # API 模式
# 或
soup = BeautifulSoup(resp.text, "html.parser")  # HTML 模式

# 第 4 步：存数据
save_csv(data)
```

**就这 4 步，三个平台都一样。** 区别只在于：
- URL 不同
- 参数格式不同
- 响应 JSON 的结构不同（需要找到数据在哪个字段里）

## 5. 为什么会被封？（反爬原理）

网站怎么识别你不是真人？

| 检测手段 | 原理 | 应对（入门级） |
|----------|------|---------------|
| **User-Agent** | 识别"你是浏览器还是脚本" | 伪造 UA Headers |
| **Cookie/Session** | 没登录态的请求可疑 | 带上 Cookie |
| **请求频率** | 1秒发100个请求，人做不到 | 加随机延时 2-5 秒 |
| **Referer** | 检查你是从哪个页面跳过来的 | 伪造 Referer Header |
| **JavaScript 渲染** | 页面内容是 JS 动态生成的 | 需要 Selenium/Playwright |
| **验证码/CAPTCHA** | 确认你是人 | 入门级：手动处理 |

**入门阶段：做好前 4 个就够了。** 不需要代理池，不需要 Selenium。

## 6. 调试技巧（最重要）

### 浏览器开发者工具 = 你的侦察兵

爬任何网站之前，先用浏览器观察：

1. **打开 DevTools**（F12）→ **Network** 标签
2. **在网页上操作**（搜索、翻页）
3. **观察请求列表**：
   - 哪个 URL 是数据接口？
   - 请求方法是 GET 还是 POST？
   - 带了哪些参数？
   - 响应是 JSON 还是 HTML？
4. **右键请求 → Copy as cURL** → 可以直接在终端测试

### 翻译成 Python

```
浏览器里看到的:
  GET https://www.ozon.ru/api/composer-api.bx/page/json/v2?url=/search/?text=耳机&page=1
  Headers: User-Agent: ..., Accept: application/json

翻译成 Python:
  resp = requests.get(
      "https://www.ozon.ru/api/composer-api.bx/page/json/v2",
      params={"url": "/search/?text=耳机&page=1"},
      headers={"User-Agent": "...", "Accept": "application/json"}
  )
```

**核心心法：浏览器里看到什么请求，Python 就构造什么请求。**

## 7. 你现有的 Ozon 代码对照

看你已有的 `ozon_search.py`，它做的事情：

```
1. create_session()         → 创建带重试的 HTTP 客户端
2. search_api(session, keyword, page)  → 构造请求 → 发请求 → 解析 JSON
3. search_html(session, keyword, page) → 备用方案，解析 HTML
4. save_csv(products)       → 存数据
5. run()                    → 串起来，循环翻页，加延时
```

这就是完整的爬虫结构。后面写沃尔玛和亚马逊，套同样的模式就行。
